Command line, terminal, command prompt, shell... these can all refer to a user interface that allows you to interact with an Operating System through built-in commands and scripts developed by yourself or a third party. *CLICK*

The Windows Shell is one graphical user interface that many people are familiar with. Finder, Dock, Mission Control, and other components make up the graphical UI "shell" for OSX. *CLICK*

The MS DOS Command Prompt, OSX Terminal, and over a dozen different Linux shells, are examples of text-based, or command-line, user interfaces. That's what we'll be talking about tonight. Specifically, I want to talk about the Linux command-line interface shell called Bash.

The Bourne-again Shell, or Bash, is one of the most common shell environments {{Citation needed}}. It is the default shell in the OSX Terminal and is available on many Linux operating systems. In Windows 10 it is even possible to install a Bash shell using the Windows Subsystem for Linux beta feature. In earlier versions of Windows, you can install Linux-like environments such as Cygwin or MinGW, which are really collections of GNU utilities running in Windows instead of on a Linux kernel.

At this point you should all have a Bash shell installed and ready to go on your computer.

Commands, Options, and Arguments

$ pass salt
$ pass --carefully salt
$ pass salt me

00:07:00

When using a shell like Bash, you type commands that tell the computer what to do, using options that tell the computer how to do it, and arguments that describe what to do it to.

For example: "pass the salt" *CLICK* might be written as "pass salt"

If I were asking my three-year-old to hand me the salt shaker, I might add the option "carefully" to the command. *CLICK*

And if I were addressing a very literal-minded person I could add a second argument, just to be explicit: "pass the salt to me" *CLICK*

After typing a command with all of the desired options and arguments, you press the enter key to execute the command.

The file systems most of us work with, and the ones we will be talking about today, are hierarchical. You might be used to seeing them displayed as a tree, such as in the Windows Explorer or OSX Finder applications. The linux tree command produces output like that shown here. A directory, or "folder," can contain files or other directories, called subdirectories. The terms "parent" and "child" are also used to describe directories in hierarchies.

00:10:00

Any time you are in a Bash shell, or other command-line environment, you will have a location, or current working directory, within the file system. The pwd command prints your current working directory. *CLICK*

To move around the file system, you use the cd command to change directories. *CLICK* The command takes an argument that indicates the directory you want to move into, and that directory becomes your current working directory. This argument can a complete path from the root of the file system (called an absolute path) where the root is represented by a slash (/) or it can be relative which is a path described from the current working directory. You can think of an absolute path as directions to a location beginning from a fixed starting point (the root of the file system) and a relative path can be thought of as directions beginning from where you are right now (the current working directory).

  • Directories in a path are separated by slashes
  • Your current working directory can be represented in a path by a "dot" or period
  • The parent of a directory in a path can be represented by two dots
  • Your home directory can be represented by a tilde character (~).

New directories can be created with the mkdir command. *CLICK* Just like cd, this takes an argument for the path of the directory to be created.

Empty directories can be removed using the rmdir command. Again, use either an absolute or relative path as the argument to indicate the directory you want to remove.

Creating files in Bash

$ touch newfile
$ echo "Hello, File!" >> greeting
$ vi newfile
$ start newfile

00:30:00

The touch command will create a new, empty file, or update last modification date and time on an existing file. This can be useful if you want to ensure a file exists with a particular name, but don't have any content to write into it yet.

The echo command takes a string of characters as an argument and outputs, by default, back on the screen. We can combine this with a pair of greater-than signs to redirect output from the screen to another destination; in this case, to a file called "greeting".

Echoing text into a file is handy, but you probably want a more fully-functional editing experience for working with text files. In Linux, common editors include vi, emacs, and nano. Much of the work editing this presentation was done using vi. We don't have time to go into too much depth with these editors, which each have their own unique keyboard commands and quirks. Fortunately, it's also fairly easy to open a graphical user interface for editing a file. In Windows you can open a file for editing using the start command, or open in OSX.
